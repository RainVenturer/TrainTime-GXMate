相关问题：

Q：默认校园网超额了咋办？

A：超额后仍可免费正常访问教学科研服务资源，其他资源进行限速管理。目前计划限速资源范围为主流视频网页、直播网站、游戏资源，后续可动态调整。

Q：免费额度能否结转？

A：不能。因业务计费规则，学生及职工每月免费的校园网使用额度，仅在当月生效，不可跨月使用；当月未使用的额度无法结转。

Q：请问有无无限量校园网服务？

A：虑到校园网部分用户仍有较大其他的需求，学校现已引入联通与电信两家运营商为教工和学生用户提供非免费或其他上网服务，用户可自由选择。详情请咨询相关运营商。

Q：必须办理学校规定的手机SIM卡吗?

A：学校从来没有规定必须办理某种手机卡，或者使用某种套餐，请大家谨防受骗！

2. 如何使用校园网?

每一位同学默认都开启了校园网服务，账号为“学号”，初始密码为身份证后六位。

宿舍中已经为大家加装了无线网络ap盒， 仅供提高网络速度和使用体验，不可连接网线。

注意：以下描述大多数内容仅涉及默认校园网账户，如果您购买了运营商服务，请看本文档第四章：运营商账号使用说明

  1. WIFI 使用方式

    在全校范围内，使用 WLAN 连接名称为stu-xdwlan/stu-wlan/xd-wlan（前者优先） 的热点，根据提示，帐号为学号，默认密码为身份证后六位即可使用 ，密码修改过的，使用修改后的密码登录。

    若连接 Wi-Fi 后没有提示需要登录，则可以手动在浏览器打开 https://10.255.44.33 或 https://w.xidian.edu.cn 进行登录。

    注意，部分用户无法网址登录的，可以尝试在保持 stu-xdwlan 连接下进入学在西电app，在弹窗中登录。

    新设备 (手机/电脑) 在首次登录校园网时，可能出现“网络延迟，需要刷新页面后重新登录”，请参考文档第16条开启无感知认证。

    使用本方式进行认证的时候，用户不能使用代理软件或浏览器开启代理功能，并且DNS必须是自动获取学校DNS，不能手动配置。

  1. 拨号使用方式

    在全校范围内，使用网线插入端口连接电脑 (例如宿舍桌下的端口)，可以新建 PPPoE 宽带链接。路由器可以将一人的校园网分享给多个设备使用。

  2. MAC认证上网

    MAC认证是校园网无线的一种认证方式，该方式可让您无需在登录过的终端上重新走一遍流程了。当终端设备第一次使用校园网无线认证成功上网后，系统会绑定你当前登录终端的MAC地址，当此终端再次登录校园无线时，不会在弹出potal页面，连上WIFI后会通过MAC地址登录绑定账号，直接上网。

    每个校园网账号可以绑定6个MAC终端（MAC地址绑定不仅记录无线终端MAC还会记录有线认证终端MAC），当记录满之后，再次登新的终端时，会随机替换已绑定MAC中的一个，用户可以通过校园网自服务平台查看、增加或解绑MAC地址，也可关闭MAC认证功能，如下图。

    关于MAC认证上网和其他事宜，请参考第十三章：开启/关闭无感认证。

4.更新 2025-03-17：运营商账号使用说明

  1. 订购产品

    运营商认证上网账号密码与校园网相同，如需订购运营商产品，需要到学校运营商处订购运营商产品，详情请咨询运营商。

  2. 产品使用

    订购产品后需要登录校园网自服务平台，打开左侧菜单栏，点击“产品-产品使用优先级”，将要使用的产品拖拽到最上面即可生效，具体页面如下图：

    所有账号默认产品是校园网，如果要使用校园网把校园网拖拽到最上面，如果要使用运营商产品，把运营商产品拖拽到最上面即可。

    注：选择使用运营商产品的用户，不管是无线还是有线认证，校园网产品要放在自服务“产品使用优先级”最下面，否则即使选择的是运营商产品，实际使用的却是校园网。

    MAC 认证用户须知：MAC认证只要在自服务上将要使用的产品拖拽到最上面，校园网产品在最下面即可。

浏览器打开 https://vpn.xidian.edu.cn/，首先点击“下载客户端”，下载安装 EasyConnect 软件，安装完成后，打开软件进行操作：

在软件的连接地址一栏内填写：vpn.xidian.edu.cn ， 点击下一步按钮：

随后填写在账号一栏填写学号，在密码一栏填写统一身份认证密码：

注意：出于安全考虑，减轻流量压力，西电 VPN 仅开放了如上图所示资源与部分公共资源的访问权限。

部分未开放服务、普通学生与教职工电脑分配到的 IP 地址、由信息网络技术中心运营的部分虚拟机的直接 IP 地址等受到学校防火墙保护，并不在 VPN 可访问范围内。详情请咨询有关人员。

PPPoE 拨号错误代码速查表

该表可辅助您快速排查错误信息，并有助于相关人员定位您的问题。

常见编号：

  • 651：调制解调器报告错误。这通常是物理层的损坏，比如网线断了，没插好，网卡烧了，网卡接触不良，机房交换机出问题等。
  • 691：因为用户名和/或密码在此域上无效，所以访问被拒绝。可能账号密码输错了，或者拨号太频繁了。
